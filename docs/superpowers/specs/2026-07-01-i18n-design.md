# ON-GI 다국어(i18n) 지원 설계

## 목적

ON-GI를 글로벌 크리스천이 사용할 수 있도록 다국어를 지원한다.
기존 기능/UI를 변경하지 않고, i18n 인프라와 언어 선택 버튼만 추가한다.

## 지원 언어

| 코드 | 언어 | 표시명 | 선정 이유 |
|------|------|--------|-----------|
| `ko` | 한국어 (default) | 한국어 | 서비스 원어 |
| `en` | 영어 | English | 세계 사용량 1위, 크리스천 인구 1위 |
| `es` | 스페인어 | Español | 중남미 크리스천 + 세계 사용량 4위 |
| `fr` | 프랑스어 | Français | 아프리카 크리스천 성장세 + 프랑스어권 |
| `pt` | 포르투갈어 | Português | 브라질 크리스천 인구 + 아프리카 |

## 기술 스택

- **라이브러리**: `next-intl` (Next.js App Router 네이티브 통합)
- **라우팅**: URL 기반 — `/ko/onboarding`, `/en/onboarding` 등
- **언어 감지**: middleware에서 `Accept-Language` 헤더 → 지원 언어 매칭 → 리다이렉트
- **기본 언어**: `ko` (매칭 실패 시 fallback)

## URL 구조

```
현재:  /onboarding, /(main)
변경:  /[locale]/onboarding, /[locale]/(main)

예시:
  /ko/onboarding  → 한국어 온보딩
  /en/onboarding  → 영어 온보딩
  /ko             → 한국어 메인
  /fr/invite/abc  → 프랑스어 초대 링크
```

## UI 변경 (최소)

### 1. 온보딩 페이지 — 하단 드롭다운
- 화면 하단에 `🌐 한국어 ▾` 드롭다운 버튼 추가
- 탭하면 5개 언어 목록이 올라옴
- 선택 시 해당 locale URL로 이동 (예: `/en/onboarding`)
- 기존 UI 요소(로고, 태그라인, 닉네임 입력, 함께하기 버튼) 변경 없음

### 2. 설정 패널 — 언어 섹션 추가
- 기존 "👤 계정" 섹션 아래에 "🌐 언어" 섹션 추가
- "앱 언어" 행 — 현재 언어 표시 (골드 뱃지) + 탭하면 선택 시트
- 선택 시 locale URL 변경 + 페이지 전환
- 기존 설정 항목(닉네임, 초대 링크, 프라이버시) 변경 없음

## 파일 변경 범위

### 신규 파일
- `src/messages/ko.json` — 한국어 번역 (현재 하드코딩된 문자열 추출)
- `src/messages/en.json` — 영어 번역
- `src/messages/es.json` — 스페인어 번역
- `src/messages/fr.json` — 프랑스어 번역
- `src/messages/pt.json` — 포르투갈어 번역
- `src/i18n/config.ts` — locale 목록, default locale 등 설정
- `src/i18n/request.ts` — next-intl 서버 요청 설정
- `src/components/ui/LanguageSelector.tsx` — 언어 선택 드롭다운 컴포넌트

### 수정 파일
- `next.config.ts` — `createNextIntlPlugin` 래핑
- `src/middleware.ts` — next-intl middleware로 교체/확장
- `src/app/layout.tsx` → `src/app/[locale]/layout.tsx` — locale 세그먼트 추가
- `src/app/(auth)/onboarding/page.tsx` → `src/app/[locale]/(auth)/onboarding/page.tsx`
- `src/app/(main)/page.tsx` → `src/app/[locale]/(main)/page.tsx`
- `src/app/invite/[code]/page.tsx` → `src/app/[locale]/invite/[code]/page.tsx`
- 하드코딩된 한국어 → `useTranslations()` 호출로 교체 (11개 파일)

### 변경하지 않는 것
- DB 스키마 — 언어는 URL로 관리, DB 컬럼 추가 없음
- Supabase 설정 — 변경 없음
- Globe/Mapbox 렌더링 로직 — 변경 없음
- 기도 시작/종료 로직 — 변경 없음
- PWA manifest — `name`/`short_name`은 유지 (PWA는 설치 시 언어 고정)

## SEO

- 각 locale별 독립 URL → 검색엔진 별도 인덱싱
- `<html lang={locale}>` 자동 설정
- OG 메타 태그의 title/description도 locale별 번역
- `alternate` hreflang 태그 자동 생성 (next-intl 기본 지원)

## 검증 방법

1. `npm run build` 통과
2. `/ko/onboarding` → 한국어 표시 확인
3. `/en/onboarding` → 영어 표시 확인
4. 온보딩 하단 드롭다운으로 언어 전환 → URL 변경 확인
5. 설정 패널 언어 변경 → URL 변경 + 전체 텍스트 전환 확인
6. 초대 링크 (`/fr/invite/abc`) → 프랑스어 OG 메타 확인
7. 브라우저 언어 영어 → 첫 방문 시 `/en`으로 리다이렉트 확인
